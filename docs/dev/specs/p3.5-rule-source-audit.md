# P3.5 Rule Source Audit / Seed Rule Review Spec

Date: 2026-06-26

Branch: `wbs/P3.5-rule-source-audit`

## Goal

P3.5는 correction rule seed가 어떤 데이터에서 왔는지 검증하고, final eval leakage 없이 안전하게
초기 seed rule만 분리한다.

이 phase는 automatic rule promotion이나 final production rule 확정 단계가 아니다. 목적은 provenance,
support, risk, candidate mode를 구조화해 이후 P4/P5/P8/P9가 같은 rule 기준을 사용할 수 있게 하는 것이다.

## Inputs

Audit input은 다음 source type을 지원한다.

| Source type | Purpose |
| --- | --- |
| `dictionary_registry` | 도메인 사전/용어 registry 기반 seed |
| `aligned_segment` | ASR output과 reference alignment에서 관찰된 오인식 pair |
| `phrase_bias_result` | phrase-bias 또는 probe 결과에서 관찰된 후보 |

P3.5 구현은 JSONL input을 기준으로 한다. 대용량 원본 데이터는 commit하지 않고, 테스트 fixture와 small sample만
commit한다.

## Audit Record Contract

각 input row는 다음 필드를 가진다.

| Field | Type | Rule |
| --- | --- | --- |
| `rule_id` | string | non-empty |
| `wrong` | string | non-empty |
| `right` | string | non-empty |
| `source_type` | enum | `dictionary_registry`, `aligned_segment`, `phrase_bias_result` |
| `provenance` | string | non-empty |
| `source_wav` | string/null | source wav reference |
| `segment_id` | string/null | source segment id |
| `split` | string | `train`, `dev`, `final_eval`, `unknown` |
| `support_count` | integer | 0 이상 |
| `is_free_talk` | boolean | free-talk source 여부 |
| `is_script_span` | boolean | script/domain span 여부 |
| `contains_number_or_pii` | boolean | 숫자/PII 민감 surface 여부 |
| `span_length_chars` | integer | wrong/right 중 긴 쪽의 문자 길이 |
| `domain_term` | string | domain term hint |

## Candidate Decision Policy

Audit decision은 다음 값을 가진다.

| Decision | Meaning |
| --- | --- |
| `safe_seed` | safe-only seed CSV에 포함 가능 |
| `review_optional` | optional rule 후보지만 safe-only seed에는 제외 |
| `disabled` | 위험하거나 leakage 가능성이 있어 disabled |

Decision rule:

- `split=final_eval`이면 `disabled`
- `support_count < 2`이면 `review_optional`
- `is_free_talk=true`이고 `is_script_span=false`이면 `review_optional`
- `contains_number_or_pii=true`이면 `disabled`
- `span_length_chars > 24`이면 `disabled`
- 위 조건에 걸리지 않으면 `safe_seed`

Candidate mode mapping:

- `safe_seed` -> `optional`
- `review_optional` -> `optional`
- `disabled` -> `disabled`

P3.5는 obligatory rule을 자동 생성하지 않는다. Obligatory promotion은 후속 review와 evaluation 후 결정한다.

## Outputs

### Audit Report

`docs/reports/probes/rule_source_audit.md`는 다음을 포함한다.

- input count
- decision별 count
- safe-only seed count
- final eval leakage count
- high-risk disabled count
- rule별 provenance table

### Seed CSV

`data/correction_rules_seed.csv`는 `safe_seed` decision만 포함한다. CSV field order는 P3
`CorrectionRule` contract와 동일하다.

### Tests

`tests/test_rule_source_audit.py`는 다음을 검증한다.

- final eval 유래 rule이 safe-only seed에서 제외된다.
- low-support rule은 optional review로 남고 safe-only seed에서 제외된다.
- 숫자/PII 또는 긴 script span rule은 disabled된다.
- safe-only CSV는 P3 `read_correction_rules_csv()`로 읽을 수 있다.
- report는 leakage count와 provenance를 포함한다.

## Non-goals

- production rule 확정
- automatic rule promotion
- LLM 기반 rule 생성
- raw audio or large alignment artifact commit

## Acceptance Criteria

- seed rule마다 provenance와 support가 기록된다.
- final eval 유래 rule은 final claim에 사용하지 않도록 표시된다.
- `safe_only` seed CSV가 생성된다.
- high-risk rule은 disabled 또는 optional 후보로만 남는다.
- `uv run --group dev pytest -q`가 통과한다.
- `uv run --group dev ruff check .`가 통과한다.
