# P3.5 Rule Source Audit / Seed Rule Review Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox
> (`- [ ]`) syntax for tracking.

**Goal:** Build a reproducible rule source audit that separates safe seed rules from leakage-prone or risky candidates.

**Architecture:** Keep audit DTO and decision logic in `src/whisper_wfst/rule_source_audit.py`. Reuse P3
`CorrectionRule` CSV IO for safe-only seed output. Provide a CLI in `scripts/audit_rule_sources.py` that reads JSONL
input, writes `data/correction_rules_seed.csv`, and writes a markdown report.

**Tech Stack:** Python 3.11+, standard library json/csv/dataclasses/pathlib, pytest, ruff.
